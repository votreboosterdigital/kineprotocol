## Corrigé

### 1. `.gitignore` — Protection `.mcp.json`
**Problème :** `.mcp.json` et sa copie backup contiennent les tokens Supabase (`sbp_`), Stripe live (`sk_live_`) et Resend (`re_`). Ces fichiers n'étaient pas trackés par git mais absents du `.gitignore`.
**Fix :** Ajout de `.mcp.json` et `.mcp*.json` dans `.gitignore`.
**Commit :** `c26174b chore: protéger .mcp.json dans .gitignore`

### 2. `PricingCTA.tsx` — Console.error client → UX propre
**Problème :** En cas d'erreur checkout Stripe non-401, `console.error('Checkout error:', data.error)` exposait potentiellement un message d'erreur serveur dans la console navigateur.
**Fix :** Suppression du `console.error`, ajout d'un état `errorMsg` avec affichage d'un message générique à l'utilisateur.
**Commit :** `01262b8 fix(pricing): remplacer console.error client par message UX propre`

### 3. `generate-protocol/route.ts` — Masquage erreur serveur
**Problème :** Le catch exposait `error.message` (potentiellement un message Prisma, Anthropic ou interne) directement dans la réponse JSON.
**Fix :** Message générique `'Erreur lors de la génération du protocole'`.
**Commit :** `7ff29e5 fix(api): masquer erreurs serveur au client + pagination exercises`

### 4. `enrich-exercise/route.ts` — Masquage erreur serveur
**Même problème** que ci-dessus.
**Fix :** Message générique `"Erreur lors de l'enrichissement de l'exercice"`.
**Commit :** (inclus dans `7ff29e5`)

### 5. `patient-version/route.ts` — Masquage erreur serveur
**Même problème** que ci-dessus.
**Fix :** Message générique `'Erreur lors de la génération de la version patient'`.
**Commit :** (inclus dans `7ff29e5`)

### 6. `exercises/route.ts` — Pagination + select explicite
**Problème :** `findMany` sans `take` ni `select` sur la table `exercises` (bibliothèque croissante, illimitée).
**Fix :** `take: 200` + `select` des colonnes nécessaires uniquement.
**Commit :** (inclus dans `7ff29e5`)
